# Next Steps

## Immediate Goal

The pure ViT baseline, projector baselines, pure ViT ONNX/OM, ViT + projector
ONNX/OM artifacts, and raw fp32 validation tensors are complete. The immediate
goal is now yellow-zone HarmonyOS device validation of the ViT + projector OM.

The expected end state is:

```text
image -> fixed 1 x 3 x 448 x 448 tensor -> vision_model -> pixel_shuffle -> mlp1 -> visual_tokens
```

## Step 1: Verify PyTorch Import

We saw a previous direct `torch` import check hang when not using the final uv-first flow. Before downloading model weights, verify the uv environment can import PyTorch reliably.

Run:

```powershell
uv run python -B -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
```

Expected:

- Command exits successfully.
- Prints the installed torch version.
- Prints `True` or `False` for CUDA availability.

If it hangs, investigate the installed wheel before continuing.

## Step 2: Confirm Transformers Remote Code Loading

InternVL uses custom Hugging Face modeling code, so loading requires `trust_remote_code=True`.
The project pins the model/code revision to:

```text
5648fa26ff23acaba53588936d9f1dfaf305f522
```

Run a small config-only check first:

```powershell
uv run python -B -c "from transformers import AutoConfig; r='5648fa26ff23acaba53588936d9f1dfaf305f522'; c=AutoConfig.from_pretrained('OpenGVLab/InternVL3_5-1B-Instruct', trust_remote_code=True, revision=r, code_revision=r); print(type(c)); print(c.vision_config.image_size)"
```

Expected:

- Config downloads.
- `vision_config.image_size` prints `448`.

## Known Risks

- The full model may be large enough to stress CPU memory during loading.
- InternVL remote code may expect specific Transformers behavior.
- PyTorch CPU inference may be slow for the 1B VLM, even though only the ViT output is used.
- If full-model loading is too heavy, the next implementation change should read only `vision_model.*` and `mlp1.*` tensors directly from safetensors instead of calling `AutoModel.from_pretrained()` for the full VLM.
- The OM files are converted but not runtime-verified; local OMG does not prove phone-side NPU placement.
